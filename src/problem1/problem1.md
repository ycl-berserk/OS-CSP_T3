## 题目背景

在操作系统的学习中，xv6 的进程管理是一个经典模型。本题要求你模拟一个简化版的进程管理器，它基于 xv6 的进程管理概念，并加入了 Linux 风格的进程树查询功能。

进程是操作系统进行资源分配和调度的基本单位。本题中，每个进程由**进程控制块 (PCB)** 表示。你需要模拟进程的创建、执行、状态转换和退出等基本生命周期。

## 题目描述

你将要实现一个进程管理器，它能处理以下操作：

- 创建新进程
- 模拟进程执行指令
- 处理时钟中断，进行进程调度
- 进程主动退出
- 查询特定进程状态
- 查询进程树关系

### 进程状态定义

在本题的简化模型中，每个进程有以下几种状态：

1. **UNUSED**: 空闲状态，进程槽未被使用。
2. **EMBRYO**: 进程正在创建中（已分配 PID，但尚未完成初始化）。
3. **SLEEPING**: 睡眠（阻塞）状态，等待某些事件（如 I/O）。
4. **RUNNABLE**: 就绪状态，进程已准备好，等待被调度执行。
5. **RUNNING**: 运行状态，进程正在 CPU 上执行。
6. **ZOMBIE**: 僵尸状态，进程已终止，但其退出状态码尚未被父进程读取。

**注意**：任何时刻，**至多只有一个**进程处于 `RUNNING`状态。

### 状态转换规则

1. **创建**：`CREATE pid ppid`
   - 分配一个空闲的 PCB，状态从 `UNUSED`变为 `EMBRYO`。
   - `pid`是指定的新进程 ID，保证不与系统中任何**非 UNUSED** 状态的进程 ID 重复。
   - `ppid`是其父进程 ID。父进程必须存在且状态不是 `UNUSED`或 `ZOMBIE`。
   - 创建成功后，新进程状态立即变为 `RUNNABLE`，并记录其进入就绪队列的时间戳。

2. **执行**：`EXEC pid instructions`
   - 只有状态为 `RUNNING`的进程可以执行指令。
   - `pid`必须与当前 `RUNNING`进程的 ID 一致。
   - `instructions`是一个正整数，表示该进程此次希望连续执行的指令条数。
   - 系统有一个**时间片** 大小 `Q`，为一个全局常量（`Q=2`）。进程最多连续执行 `min(instructions, Q)`条指令。
   - 执行分为两种情况：
     a) `instructions <= Q`：进程能一次执行完所有指令。执行后，**状态保持不变**（仍是 `RUNNING`）。
     b) `instructions > Q`：进程用尽时间片，只执行了 Q 条指令。触发**时钟中断**。进程状态由 `RUNNING`变为 `RUNNABLE`，记录新的时间戳。系统需要进行一次进程调度。

3. **调度**
   - 当 CPU 空闲（无 `RUNNING` 进程）且 `RUNNABLE` 队列不为空时，触发调度。
   - **调度算法**：选择**最早进入 `RUNNABLE`状态**的进程。如果多个进程进入时间相同，则选择 **PID 较小**的那个。
   - 被选中的进程状态由 `RUNNABLE`改为 `RUNNING`。

4. **退出**：`EXIT pid exit_code`
   - 只有状态为 `RUNNING`的进程可以执行此操作。
   - `pid`必须与当前 `RUNNING`进程的 ID 一致。
   - 使进程 `pid`进入 `ZOMBIE`状态，并记录其退出状态码 `exit_code`以及变为僵尸状态的时间戳。
   - **唤醒父进程**：检查该进程的父进程是否正处于 `SLEEPING`状态。如果是，则将父进程的状态从 `SLEEPING`改为 `RUNNABLE`，记录其进入就绪队列的时间戳。
   - 退出操作完成后，当前 CPU 空闲，立即进行一次调度。

5. **等待**：`WAIT pid`
   - 只有状态为 `RUNNING`的进程可以执行 `WAIT`。
   - `pid`必须与当前 `RUNNING`进程的 ID 一致。
   - 该操作会立即检查当前进程是否有**已处于 `ZOMBIE`状态的直接子进程**。
   - **回收策略**：选择**最早变为 `ZOMBIE`状态**的直接子进程。如果时间相同，选择 **PID 较小**的那个。
   - **如果有僵尸子进程**：回收该子进程（状态置为 `UNUSED`），输出该子进程的 `exit_code`。父进程状态保持 `RUNNING`。
   - **如果没有僵尸子进程**：当前 `RUNNING`进程状态变为 `SLEEPING`，输出字符串 `SLEEP`。之后立即触发一次调度（CPU 空闲）。

6. **查询进程状态**：`QUERY_PROC pid`
   - 查询指定 `pid`进程的当前状态（英文大写）。

7. **查询进程树**：`QUERY_TREE pid`
   - 以 `pid`对应的进程为根节点，按层序遍历输出其所有**有效**后代进程的 PID。
   - **有效进程**定义为状态不是 `UNUSED` 的进程（包含 `ZOMBIE` 进程）。
   - 输出格式：先输出根节点 PID，然后按层序每行输出一层。同一层的 PID 按从小到大顺序排列，用空格分隔。如果除了根节点外没有其他有效进程，则只输出一行（根节点 PID）。

### 初始条件

- 初始时，系统中存在一个 PID 为 1 的进程，状态为 `RUNNING`。它是所有进程的祖先。
- 时间片大小 `Q = 2`。
- 进程 ID (`pid`) 为不超过 10^9 的正整数。

## 输入格式

从标准输入读入数据。

第一行，一个正整数 `n`，表示操作条数。

接下来 `n`行，每行描述一个操作。操作格式如下：

- `CREATE pid ppid`
- `EXEC pid instructions`（`instructions`为正整数）
- `EXIT pid exit_code`（`exit_code`为整数）
- `WAIT pid`
- `QUERY_PROC pid`
- `QUERY_TREE pid`

## 输出格式

输出到标准输出。

对于每一个操作，根据操作类型输出相应结果：

- 对于 `CREATE`操作：输出一行，内容为 `OK`或 `ERROR`。
- 对于 `EXEC`操作：输出一行，包含两个整数，用空格分隔，分别为：
  1. 该进程**实际执行**的指令条数。
  2. 执行此操作后，系统中处于 `RUNNING`状态的进程 PID。如果没有进程处于 `RUNNING`，则输出 `0`。
- 对于 `EXIT`操作：输出一行，为一个整数，表示执行此操作后，系统中处于 `RUNNING`状态的进程 PID。如果没有进程处于 `RUNNING`，则输出 `0`。
- 对于 `WAIT`操作：输出一行。
  - 如果成功回收了一个僵尸子进程，则输出该子进程的 `exit_code`。
  - 如果当前进程没有僵尸子进程，导致自身进入 `SLEEPING`，则输出字符串 `SLEEP`。
- 对于 `QUERY_PROC`操作：输出一行，为指定进程的当前状态。
- 对于 `QUERY_TREE`操作：按题目描述格式输出进程树。

## 样例说明

1.  `EXEC 1 5`: P1 时间片用完 -> P2 运行。
2.  `EXEC 2 1`: P2 执行 1 条 -> P2 仍运行。
3.  `EXIT 4 100`: P4 非运行态，操作失败，输出 ERROR。
4.  `WAIT 2`: P2 无僵尸子进程 -> P2 睡眠 -> P3 运行。
5.  `EXEC 3 1`: P3 执行 1 条 -> P3 仍运行。
6.  `EXIT 3 0`: P3 退出 -> 僵尸。P1 此时是 RUNNABLE (之前 EXEC 1 5 后变就绪)。P1 运行。
7.  `WAIT 1`: P1 回收 P3 -> 输出 0。P1 继续运行。
8.  ...后续操作依此逻辑推演。

## 数据范围

- 对于 30% 的数据，没有 `WAIT`操作，且操作数 n≤1000。
- 对于 60% 的数据，操作数 n≤5000。
- 对于 100% 的数据，操作数 n≤105，进程数不超过 5×104。

## 提示

- 本题为大型模拟题，需要仔细阅读题目描述，理解进程状态转换的每一个细节。
- 注意 `EXIT` 操作不仅改变当前进程状态，还可能唤醒父进程。
- 调度算法严格遵循“先入先出（FIFO）”原则，需维护好进程进入 `RUNNABLE` 状态的时间戳。
